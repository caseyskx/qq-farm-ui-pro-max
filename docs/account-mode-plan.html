一、需求分析与核心规则

📌 核心设计原则

本系统基于以下用户确认的核心规则进行设计，确保功能符合实际使用场景。

1.1 模式切换覆写范围

✅ 覆写配置项

  • 偷取功能开关（friend_steal）
  • 捣乱功能开关（friend_bad）
  • 好友黑名单（friendBlacklist）
  • 收获延迟设置（harvestDelay）
  • 帮忙功能开关（friend_help）

🔒 保留配置项

  • 种植策略（plantingStrategy）
  • 偏好种子（preferredSeedId）
  • 任务自动化（task）
  • 邮箱自动化（email）
  • Interval 间隔设置
  • 好友静默时段

1.2 风险规避模式黑名单规则

🛡️ 自动黑名单计算逻辑

符合用户预期：自动计算"除系统内账号之外的好友全部加黑名单"

  • 大号：黑名单 = 所有好友 - 系统内所有小号
  • 小号：黑名单 = 所有好友 - 系统内所有大号
  • 每次好友列表变更时自动更新黑名单
  • 支持手动调整后锁定，不再自动覆盖

1.3 小号收获延迟设置

⏱️ 延迟计算公式

实际延迟 = 基准值 + 随机值 (±30 秒)

默认基准值：4 分钟（240 秒）

随机范围：±30 秒（防检测）

最终范围：210 秒 ~ 270 秒（3.5~4.5 分钟）

🎲 防检测机制

  • 每次收获独立计算随机值
  • 使用加密安全的随机数生成器
  • 记录每次延迟日志便于审计
  • 可配置是否启用随机浮动

1.4 大号唯一性规则

👑 单一大号限制

  • 强制约束：每个账号下的所有农场账户只能有 1 个大号模式
  • 系统检查：切换大号模式时自动检测是否已存在大号
  • 自动降级：设置新大号时，原大号自动降级为普通号
  • 前端提示：切换时弹出确认对话框说明影响范围

二、账号模式详细设计

👑 大号模式（Normal）

  • 偷取功能：开启
  • 帮忙功能：开启
  • 捣乱功能：开启
  • 收获延迟：无
  • 黑名单：用户自定义
  • 种植策略：保留用户设置
适用场景：主力账号，功能完整，适合信任度高的大号

🐭 小号模式（Small）

  • 偷取功能：关闭
  • 帮忙功能：开启
  • 捣乱功能：关闭
  • 收获延迟：240±30 秒
  • 黑名单：自动管理（可选）
  • 种植策略：保留用户设置
适用场景：辅助账号，延迟收获让大号先偷，降低风险

🛡️ 风险规避模式（Risk Avoidance）

  • 偷取功能：条件开启
  • 帮忙功能：开启
  • 捣乱功能：关闭
  • 收获延迟：可配置
  • 黑名单：全自动管理
  • 种植策略：保留用户设置
适用场景：批量运营，最大化降低封号风险

2.1 风险规避模式子角色

配置项 主账号（Main） 小号（Small）
偷取好友范围 仅小号 不偷取
帮忙对象 所有好友 仅主账号 + 其他小号
黑名单规则 除小号外全部拉黑 除主账号外全部拉黑
捣乱功能 关闭 关闭
收获延迟 无 可配置

4.1 新增服务层

/** * 账号模式管理器 * 文件：core/src/services/account-mode-manager.js */ class AccountModeManager { // 模式枚举 static MODES = { NORMAL: 'normal', SMALL: 'small', RISK_AVOIDANCE: 'risk_avoidance' }; // 角色枚举（风险规避模式） static ROLES = { MAIN: 'main', SMALL: 'small' }; /** * 切换账号模式 * @param {number} accountId - 账号 ID * @param {string} newMode - 新模式 * @param {object} options - 选项 */ async switchMode(accountId, newMode, options = {}) { // 1. 验证大号唯一性 if (newMode === this.MODES.NORMAL) { await this.ensureOnlyOneNormalMode(accountId); } // 2. 获取当前配置 const currentConfig = await this.getCurrentConfig(accountId); // 3. 计算新模式配置（只覆写指定项） const newConfig = await this.calculateModeConfig(accountId, newMode, options); // 4. 应用配置变更 await this.applyConfigChanges(accountId, newConfig, { preserveKeys: ['plantingStrategy', 'preferredSeedId', 'task', 'email'] }); // 5. 记录日志 await this.logModeSwitch(accountId, currentConfig.mode, newMode, options); // 6. 风险规避模式：自动管理黑名单 if (newMode === this.MODES.RISK_AVOIDANCE) { await this.autoManageBlacklist(accountId); } return { success: true, config: newConfig }; } /** * 计算收获延迟（小号模式） * @param {number} accountId * @returns {number} 延迟毫秒数 */ async calculateHarvestDelay(accountId) { const modeConfig = await this.getModeConfig(accountId); if (modeConfig.mode !== this.MODES.SMALL) { return 0; // 非小号模式无延迟 } const baseDelay = modeConfig.harvest_delay_base || 240; // 默认 240 秒 const randomRange = modeConfig.harvest_delay_random || 30; // 默认±30 秒 // 计算随机值：base ± random const randomValue = (Math.random() * 2 - 1) * randomRange; const finalDelay = baseDelay + randomValue; // 记录日志 logger.info(`[小号模式] 账号${accountId} 收获延迟：${finalDelay.toFixed(1)}秒 (基准:${baseDelay}±${randomRange})`); return finalDelay * 1000; // 转换为毫秒 } /** * 风险规避模式：自动管理黑名单 */ async autoManageBlacklist(accountId) { const modeConfig = await this.getModeConfig(accountId); // 如果黑名单已锁定，跳过自动管理 if (modeConfig.blacklist_locked) { logger.info(`[风险规避] 账号${accountId} 黑名单已锁定，跳过自动管理`); return; } const allFriends = await this.getAllFriends(accountId); const systemAccounts = await this.getSystemAccounts(); const blacklist = []; if (modeConfig.role === this.ROLES.MAIN) { // 主账号：黑名单 = 所有好友 - 系统内所有小号 const smallAccounts = await this.getLinkedSmallAccounts(accountId); const smallGids = new Set(smallAccounts.map(a => a.gid)); blacklist.push( ...allFriends .filter(f => !smallGids.has(f.gid)) .map(f => f.gid) ); } else if (modeConfig.role === this.ROLES.SMALL) { // 小号：黑名单 = 所有好友 - 系统内所有主账号 const mainAccounts = await this.getLinkedMainAccounts(accountId); const mainGids = new Set(mainAccounts.map(a => a.gid)); blacklist.push( ...allFriends .filter(f => !mainGids.has(f.gid)) .map(f => f.gid) ); } // 更新黑名单 await setFriendBlacklist(accountId, blacklist); logger.info(`[风险规避] 账号${accountId} 黑名单已更新：${blacklist.length}人`); } }

4.2 Repository 层扩展

/** * 账号仓库扩展方法 * 文件：core/src/repositories/account-repository.js */ class AccountRepository { /** * 检查是否已存在大号模式账号 */ async hasNormalModeAccount(excludeAccountId = null) { const pool = await getPool(); const sql = excludeAccountId ? 'SELECT COUNT(*) as count FROM account_configs WHERE account_mode = ? AND account_id != ?' : 'SELECT COUNT(*) as count FROM account_configs WHERE account_mode = ?'; const params = excludeAccountId ? ['normal', excludeAccountId] : ['normal']; const [[{ count }]] = await pool.execute(sql, params); return count > 0; } /** * 获取所有系统内账号（用于黑名单计算） */ async getSystemAccounts() { const pool = await getPool(); const [[rows]] = await pool.execute(` SELECT a.id, a.uin, a.nick, a.name, c.account_mode, c.is_primary_account FROM accounts a LEFT JOIN account_configs c ON a.id = c.account_id WHERE a.running = 1 ORDER BY c.is_primary_account DESC, a.created_at ASC `); return rows; } /** * 获取关联账号列表 */ async getLinkedAccounts(accountId, relationshipType = null) { const pool = await getPool(); let sql = 'SELECT * FROM account_relationships WHERE account_id = ?'; let params = [accountId]; if (relationshipType) { sql += ' AND relationship_type = ?'; params.push(relationshipType); } const [[rows]] = await pool.execute(sql, params); return rows; } /** * 建立账号关联 */ async linkAccounts(mainAccountId, smallAccountId) { const pool = await getPool(); await pool.execute(` INSERT IGNORE INTO account_relationships (account_id, related_account_id, relationship_type) VALUES (?, ?, 'main_to_small'), (?, ?, 'small_to_main') `, [mainAccountId, smallAccountId, smallAccountId, mainAccountId]); } /** * 解除账号关联 */ async unlinkAccounts(mainAccountId, smallAccountId) { const pool = await getPool(); await pool.execute(` DELETE FROM account_relationships WHERE (account_id = ? AND related_account_id = ?) OR (account_id = ? AND related_account_id = ?) `, [mainAccountId, smallAccountId, smallAccountId, mainAccountId]); } }

五、核心功能实现逻辑

5.1 模式切换配置计算

/** * 计算新模式配置（只覆写指定项） */ async function calculateModeConfig(accountId, newMode, options = {}) { const currentConfig = await getConfigSnapshot(accountId); // 深拷贝当前配置 const newConfig = JSON.parse(JSON.stringify(currentConfig)); // 更新模式标识 newConfig.accountMode = newMode; switch (newMode) { case 'normal': // 大号模式 newConfig.automation.friend_steal = true; newConfig.automation.friend_help = true; newConfig.automation.friend_bad = true; // 保留原有黑名单 break; case 'small': // 小号模式 newConfig.automation.friend_steal = false; newConfig.automation.friend_help = true; newConfig.automation.friend_bad = false; // 保留原有黑名单或清空（根据选项） if (options.clearBlacklist) { newConfig.friendBlacklist = []; } break; case 'risk_avoidance': // 风险规避模式 newConfig.automation.friend_help = true; newConfig.automation.friend_bad = false; // 偷取功能根据角色决定 if (options.role === 'main') { newConfig.automation.friend_steal = true; } else if (options.role === 'small') { newConfig.automation.friend_steal = false; } // 黑名单将在 applyConfigChanges 后自动管理 break; } // 应用延迟设置（仅小号模式） if (newMode === 'small') { newConfig.modeSettings = { ...newConfig.modeSettings, harvestDelayBase: options.harvestDelayBase || 240, harvestDelayRandom: options.harvestDelayRandom || 30, }; } return newConfig; } /** * 应用配置变更（保留指定配置项） */ async function applyConfigChanges(accountId, newConfig, options = {}) { const currentConfig = await getConfigSnapshot(accountId); // 需要保留的配置键 const preserveKeys = options.preserveKeys || [ 'plantingStrategy', 'preferredSeedId', 'task', 'email', 'intervals', 'friendQuietHours' ]; // 保留用户自定义配置 for (const key of preserveKeys) { if (currentConfig.hasOwnProperty(key)) { newConfig[key] = currentConfig[key]; } } // 应用新配置 await applyConfigSnapshot(newConfig, { accountId }); }

5.2 大号唯一性检查

/** * 确保大号模式唯一性 */ async function ensureOnlyOneNormalMode(targetAccountId) { const hasExistingNormal = await accountRepository.hasNormalModeAccount(targetAccountId); if (!hasExistingNormal) { return; // 不存在其他大号，直接通过 } // 查找现有的大号账号 const systemAccounts = await accountRepository.getSystemAccounts(); const existingNormalAccount = systemAccounts.find( a => a.account_mode === 'normal' && a.id !== targetAccountId ); if (!existingNormalAccount) { return; } // 抛出异常，由前端处理降级逻辑 throw new Error(`已存在大号模式账号：${existingNormalAccount.name} (ID: ${existingNormalAccount.id})`); } /** * 降级现有大号（切换为新大号时调用） */ async function downgradeExistingNormal(excludeAccountId) { const systemAccounts = await accountRepository.getSystemAccounts(); const existingNormal = systemAccounts.find( a => a.account_mode === 'normal' && a.id !== excludeAccountId ); if (existingNormal) { // 降级为普通模式（非小号） const currentConfig = await getConfigSnapshot(existingNormal.id); currentConfig.accountMode = 'normal'; // 保持 normal 但标记为非主账号 await applyConfigSnapshot(currentConfig, { accountId: existingNormal.id }); logger.info(`大号降级：账号${existingNormal.name} 已降级为普通号`); } }

十一、技术难点与解决方案

🔒 大号唯一性约束

难点：并发切换时可能出现多个大号

解决方案：

  • 数据库唯一索引约束
  • 分布式锁（Redis）
  • 事务隔离级别调整
  • 前端预检查 + 后端最终检查

🎲 随机延迟可重现性

难点：需要审计追踪但又要随机

解决方案：

  • 记录每次随机种子值
  • 延迟日志包含计算过程
  • 可配置是否启用随机
  • 审计模式可查看历史延迟

🛡️ 黑名单自动管理冲突

难点：自动管理与手动调整冲突

解决方案：

  • 黑名单锁定机制
  • 手动调整后提示是否锁定
  • 解锁后重新自动计算
  • 变更记录可追溯

⚡ 批量操作性能

难点：1000+ 账号批量切换性能

解决方案：

  • 分批处理（每批 50 个）
  • 并发控制（最大 5 并发）
  • 进度条实时反馈
  • 失败重试机制

12.2 集成测试场景

🎯 端到端测试流程

  1. 创建 3 个测试账号（A、B、C）
  2. 设置 A 为大号模式
  3. 设置 B、C 为小号模式
  4. 验证 B、C 收获延迟生效
  5. 设置 A、B、C 为风险规避模式
  6. 验证 A 的黑名单自动排除 B、C
  7. 验证 B 的黑名单自动排除 A
  8. 批量切换 B、C 回大号模式
  9. 验证配置保留情况
