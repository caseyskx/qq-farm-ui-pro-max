# 微信头像与三种扫码影响分析

## 一、三种微信扫码方式概览

| 平台 | 说明 | 头像来源 | 当前实现 |
|------|------|----------|----------|
| **wx** | 微信小程序（第三方聚合 API `api.aineishe.com`） | 依赖第三方 API 返回 | **返回空字符串**，未从 API 提取 |
| **wx_ipad** | 微信 iPad 协议（Ipad860 本地服务） | `LoginCheckQR` → `headImgUrl` / `HeadUrl` | ✅ 已扫码待确认返回 `avatar`，登录成功返回 `avatar` |
| **wx_car** | 微信车机协议（Ipad860 本地服务） | 同 wx_ipad | ✅ 同上 |

## 二、各平台头像数据流

### 1. wx（微信小程序 - 第三方 API）

- **创建二维码**：`/api/qr/create` → 调用 `wxApiUrl?action=getqr`
- **轮询检测**：`/api/qr/check` → `action=checkqr` 获取 wxid → `action=jslogin` 获取 code
- **头像**：`admin.js` 第 1343 行硬编码 `avatar: ''`，**未从 `wxCheckRes` 或 `wxLoginRes` 提取**
- **影响**：wx 扫码登录的账号**永远没有头像**，前端会降级到 QQ 默认头像（若 uin 存在）或默认图标

### 2. wx_ipad / wx_car（Ipad860 协议）

- **已扫码待确认**（status=1）：`data.headImgUrl` → 返回 `avatar: HeadImgUrl`
- **登录成功**：`data.HeadUrl` 或 `data.acctSectResp` 相关字段 → 返回 `avatar: headUrl`
- **影响**：有头像，但微信 CDN 头像 URL 可能带时效 token，**存在过期失效风险**

### 3. qq（QQ 扫码）

- 使用 QQ 头像 API：`https://q1.qlogo.cn/g?b=qq&nk=${uin}&s=100`
- **稳定**：QQ 头像 URL 按 uin 生成，无时效问题

## 三、微信头像 URL 失效问题

- 微信头像通常来自 `wx.qlogo.cn` 或类似 CDN，URL 可能包含鉴权参数，**有过期时间**
- 失效后：前端 `@error` 会隐藏破损图，但不会自动降级到平台默认头像
- 建议：增加降级逻辑（见下文实现）

## 四、优化建议与实现

1. **URL 失效降级**：头像加载失败时，按 `platform` 显示平台默认头像（QQ 用 qlogo，微信用默认图标）
2. **多平台头像策略**：抽象 `getAvatarUrl(acc)`，按 `platform` + `avatar` + `uin` 生成最终 URL
3. **缓存**：可选方案
   - 前端：`sessionStorage` 缓存头像 URL（注意微信 URL 可能带时效，缓存意义有限）
   - 后端：代理下载头像并转存为本地/OSS，返回稳定 URL（需额外开发）
4. **wx 平台**：若第三方 API 返回头像字段，需在 `admin.js` 中提取并传递
