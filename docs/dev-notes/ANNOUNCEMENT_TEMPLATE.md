# 更新公告模板（精简统一版）

## 1) 标题模板

`YYYY-MM-DD 主题 前端：vX.Y.Z 后端：vA.B.C`

示例：
`2026-03-09 访客链路稳健化 前端：v4.5.12 后端：v4.5.12`

## 2) 内容模板（3~6 条）

- `[模块] 做了什么；带来什么结果。`
- `[模块] 做了什么；带来什么结果。`
- `[模块] 做了什么；带来什么结果。`
- `[修复] 修复什么问题；影响范围。`
- `[验证] 列出通过的关键检查命令。`

## 3) 写作约束

- 一版一个标题；同版本“补充”并入原条目，不新增同版本标题。
- 每条只写一个主语义，避免一条塞多个从句。
- 尽量使用“动作 + 结果”句式，少写背景故事和宣传措辞。
- 建议单条 40~65 字，优先保证可扫描。
- 术语统一：`SyncAll`、`GetAll`、`Webhook`、`auto` 等保持固定写法。

## 4) 禁止项

- 禁止重复标题（同日期同版本出现两次）。
- 禁止 `-xxx`（短横线后无空格）格式。
- 禁止仅写“优化若干”“修复一些问题”这类不可审计描述。

## 5) 发布前自检清单

- [ ] 标题是否唯一（同版本无重复）
- [ ] 条目是否 3~6 条且可扫描
- [ ] 是否包含至少 1 条“修复/风险控制”
- [ ] 是否包含“验证”条目（如构建/语法检查）
- [ ] 关键名词写法是否统一
- [ ] 已执行 `pnpm check:announcements` 且结果为 0 error
