3. 审计发现与高危修复 (v3.9.10)

⚠️ 网络层: 令牌桶饿死防护机制 ReferenceError 崩溃隐患 已修复 (v3.9.10)

审计溯源: 在 core/src/utils/network.js 的双队列防饿死安全机制中（drainQueue 方法），第 127 行存在严重的未声明变量调用隐患。

技术详情:

if (urgentConsecutiveCount >= MAX_CONSECUTIVE_URGENT && normalQueue.length > 0)

此前在进行双队列升级时，虽然写了上述逻辑限制高优抢收队列长期霸占网关，但通篇并未声明 let urgentConsecutiveCount 与 const MAX_CONSECUTIVE_URGENT。一旦网络中触发防偷抢收造成加急队列写入（urgentQueue.length > 0），Node.js V8 引擎将直接抛出 ReferenceError: urgentConsecutiveCount is not defined 导致微任务异常甚至服务整体崩溃退出。

修复成果: 已在此文件 88 行补正变量声明，彻底根除宕机风险，确保了紧急与普通防卡死队列的丝滑转盘。
