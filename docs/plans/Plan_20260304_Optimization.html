🚀 系统健壮性与性能优化计划 (V4.1)

1. ⏱️ 增强时间参数库：风险阻断与安全边界 (P0)

防封禁安全 当前隐患：全局时间参数暴漏给前端后，普通用户可能出于“提速”的本能，将 `farmCycleInterval` (循环检测) 或 `friendPatrolInterval` (好友巡查) 设置到极端危险值（如1秒），高频恶意轮询将直接触发腾讯接口的 `1002003` 防风控拦截，导致账号大规模被办。

应对策略 (Mitigation Strategy)

  • 后端校验硬保底 (Runtime Guard)：修改 `core/src/controllers/admin.js` 中的 /api/admin/time-parameters 保存接口，强制限定最小安全下限（例如：farmCycleInterval 必须 ≥ 30000ms）。
  • 权限分发控制 (Role-Based Ranges)：仅对 role === 'admin' 的所有者开放完全自定义权限（不设最低限或只设极低的 100ms 兜底），而对 user 角色则实行严苛的安全白名单管控。
  • 前端限制交互 (UI Safeguards)：将 `Settings.vue` 中涉及请求频率的参数从无极 `` 修改为更受控的 <BaseSlider>，或加上带有显著警示颜色的输入验证反馈。

2. 📉 Dashboard 日志加载性能：虚拟化与削峰 (P1)

前端性能优化 当前隐患：由于 WebSocket 会在上游连接瞬间下发 limit: 100 的多模块全量日志。若短时间内进行重连（尤其是开启了密集型日志模式），几百上千条 DOM 节点将使 Vue 进行巨量渲染计算。随着页面常驻挂机，累积的内存压力足以使得含有毛玻璃动效的 Dashboard 面页掉帧卡顿。

应对策略 (Mitigation Strategy)

  • 初始缓存减压 (Snapshot Trimming)：在 `admin.js` 中将初始 WebSocket 连接快照推送的 limit: 100 降低为最精华的 30-50 条，保障首屏秒开。
  • 淘汰算法前置 (Deduplication & Capping)：优化 `status.ts` 下针对于 `accountLogs` 缓存池的 LRU 管理。目前虽然能清点老旧数据，但当数据峰值爆发时，仍有高消耗。
  • DOM虚拟滚动降级 (DOM Recycling)：将原生的 v-for 日志流标签层级重构，移除不必要的响应式追踪（使用 `Object.freeze` 脱库），或者评估引入 `vue-virtual-scroller` 的可行性，维持界面渲染永远仅对齐可视窗口范围内的数十行节点。

3. 🧩 防过渡组件奔溃：Teleport 隔离法 (P2)

工程健壮性 当前隐患：如之前所述，Vue 3 的 <Transition> 容器极度娇贵，任何试图往根组件（比如 `Dashboard.vue` 的底层）追加功能弹窗、提示气泡（Siblings Fragment）的修改，都会引发路由卡死白屏风暴。

应对策略 (Mitigation Strategy)

  • 封装全局 Teleport：制定开发规范，未来所有属于系统悬浮层级别的组件（<ConfirmModal>、<Toast> 等），内部的挂载模板均由 <Teleport to="body"> 包裹。
  • DOM结构防呆：一旦实施 Teleport，不论开发者误将弹窗代码插在 vue 的哪个层级（包括写成平行兄弟节点），在真正渲染时，Vue 都会把他们强行摘取放置在 <body> 的末尾，彻底斩断对路由页面内“单根结构”的威胁。
  • 深度审计与规整：审查 `Friends.vue`、`Accounts.vue` 以及 `Workflow.vue` 等现行文件，将所有的弹出蒙层统一替换，避免下次接手重构时再次“踩坑”。
