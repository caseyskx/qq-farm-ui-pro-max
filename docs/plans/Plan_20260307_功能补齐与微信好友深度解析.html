功能补齐与微信好友深度解析 (Plan_20260307)

基于此前的三版本基准对比报告，针对用户提出的“微信好友无法获取”以及“通告单独管理功能”等痛点进行深度解析，并制定原子级实施计划。

深度解析一：微信好友列表获取差异

现状描述： 用户当前程序（副本）无法成功获取微信好友列表，而“偷菜版”可以。

源码比对结果：

  • 偷菜版： 强制使用 GetAllFriendsRequest 获取好友，不再区分平台。代码极其纯粹：
    sendMsgAsync('gamepb.friendpb.FriendService', 'GetAll', body)
  • 当前程序： 在 friend-actions.js 中有完整的 SyncAll 降级 GetAll 的逻辑。其针对微信会走到 GetAll。

核心断症（为什么我们的不工作？）：

由于代码层面的 Protobuf 请求构造完全一致，出现数据空置或失败的根本原因通常是以下三种：

  1. 解析缓存问题： 微信协议层返回 replyBody 后，如果我们的 types.GetAllFriendsReply.decode(replyBody) 反序列化结构在长期的重构中由于 proto 文件的变动导致字段丢失。
  2. 平台标识问题 (Platform)： 尽管有 WeChatPlatform.js，若因为前端传值或某些缓存导致 platformInst.allowSyncAll() 判断异常，可能走了 SyncAll（引发 1000020 错误），降级时 GetAll 收到了上下文污染。
  3. 微信环境强验证： 微信农场需要特定的 Header 或登陆态。若偷菜版处理了额外的登陆验证而我们遗漏了。

解决方案策略：

彻底重构并强化 GetAll 后备方案，增加“强制使用兼容模式(GetAll)”的前端开关，跳过任何平台猜测，并且在日志中原样 Dump 返回值以便诊断。

深度解析二：公告展示与管理功能

现状描述： 偷菜版有一个独立的“全服公告”管理菜单及弹窗，用于管理员在页面直接向所有用户广播信息；当前程序无此功能。

架构逆向分析 (基于偷菜版)：

  • 存储： SQLite的 announcements 表（包含 title, content, updated_at）。
  • 接口： GET /api/announcement (公开), PUT /api/announcement (仅限管理员)。
  • 展示： Vue 前端 AnnouncementDialog.vue，并且利用 localStorage('announcement_seen_id') 避免弹窗骚扰。管理员可在弹窗直接编辑保存。

实施策略：

利用当前程序更成熟的 MySQL + Redis 设施，实现一个更具弹性的公告系统。前台右上角 Navbar 增加 🔔 小喇叭带红点，点击展示；后台 Settings 提供富文本编辑器或专用管理面板。

实施阶段规划 (Atomize Tasks)

Phase 1: 微信好友深度修复与强兼容模式

高优先级 | 异常修复
  • [ ] 修改 friend-actions.js：在 getAllFriends 增加极致调试日志（记录 decode 前的 buffer 长度）。若 GetAll 返回好友数为0，记录更详细的错误事件。
  • [ ] 新增强制配置：在用户模型 (store.js / MySQL) 中新增字段 force_get_all (布尔值)。
  • [ ] 更新前端设置：在 "高级设置" 中增加开关：「兼容模式：强制使用 GetAll 接口获取好友（针对微信没好友的情况）」，选中后无论什么平台均强走 GetAll。

Phase 2: 系统公共公告管理系统实施 (System Announcement)

新功能 | 全局广播
  • [ ] 数据库建表 (MySQL)：在初始化脚本 (database-init.js) 中增加 system_announcements 表 (id, title, content, enabled, updated_at)。
  • [ ] 后端 API (api/routes.js)：
    • GET /api/system/announcement => 所有人可读，取最新启用的一条。
    • PUT /api/system/announcement => 校验 JWT 的 admin 角色，执行更新。
  • [ ] 前端读取与展示：开发组件 AnnouncementModal.vue，利用 localStorage 存储最后阅读时间（如有新公告显示红点并自动弹窗一次）。在 Navbar 或主界面增加入口。
  • [ ] 前端管理编辑：开发 AnnouncementManager.vue （限管理员访问），允许管理公告内容与发布状态。

Phase 3: 补齐其他剩余高优功能 (对比报告遗留)

  • [ ] 点券自动购买有机肥：在 mall.js 中加入自动使用点券兑换有机肥的业务逻辑。
  • [ ] 跳过白萝卜配置：在 friend-decision.js 增加过滤逻辑，并在前端 StealSettings.vue 加入「偷菜忽略白萝卜」开关。
