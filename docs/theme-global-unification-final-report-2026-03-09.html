主题全局统一最终报告

日期：2026-03-09（Asia/Shanghai）

目标：在不改业务逻辑的前提下，完成深色/浅色与主题色的语义化统一，减少硬编码、提升一致性与可维护性。

核心成果

  • 完成 主题 token 运行时框架：web/src/theme/tokens.ts、web/src/theme/apply-theme.ts。
  • 完成 全局语义变量扩展：--ui-bg-* / --ui-border-* / --ui-text-* / --ui-status-* / --ui-shadow-*。
  • 完成 基础组件色板统一：Button/Input/Select/Switch/Toast。
  • 完成 高频页面收口：Accounts、Friends、FarmTools、BagPanel、Settings。
  • 完成 iframe 页面（FarmTools）注入样式由绝对色改为语义变量驱动。

关键原则落地

  • 主题色：统一来源于 --color-primary-*，避免局部独立主色。
  • 文字色：统一到 --ui-text-1/2/3，降低明暗模式反转风险。
  • 状态色：统一到 --ui-status-success/warning/danger/info。
  • 面板色：统一到 --ui-bg-surface / --ui-bg-surface-raised。
  • 阴影：统一到 --ui-shadow-*，发光强度按语义层级控制。

残留扫描（当前快照）

统计口径：匹配 #hex / rgb(a) / shadow-[...] / text-shadow，用于定位仍可收敛项。

验收清单（建议）

  • 浅色 + default/cyber/elegant/ocean/sakura：检查导航、卡片、按钮、状态提示是否同一视觉层级。
  • 深色 + default/cyber/elegant/ocean/sakura：检查文本对比度、边框可见性、禁用态辨识度。
  • FarmTools iframe：切换主题时确认背景、按钮、徽章、表格条纹无白底闪烁。
  • Settings 预览区域：主题套装卡、背景范围卡、工作区视觉预设卡的高亮与阴影强度一致。
  • BagPanel 详情弹窗：遮罩层、主卡片、图片阴影、按钮激活态在浅/深色下都不过曝。
