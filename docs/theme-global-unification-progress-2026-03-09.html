主题全局统一进度报告

生成时间：2026-03-09（Asia/Shanghai）

目标：在不改业务逻辑前提下，建立并落地统一的深/浅色与主题色语义体系，降低硬编码颜色和页面风格漂移。

本轮已完成

  • 完成 BagPanel.vue 大量自定义样式 token 化：按钮、输入、tab、筛选、卡片、详情弹窗、状态胶囊。
  • 完成 Settings.vue 页面级语义覆写层：text/border/bg/hover 统一到 --ui-* 变量。
  • 完成 Settings.vue 状态语义映射：success/danger/info/warning 文本与背景统一。
  • 完成 FarmTools.vue 首轮样式统一：兜底色与局部控件样式改为语义 token。

当前收益

  • 浅色模式下不再依赖大量暗色透明白字方案，阅读对比度明显稳定。
  • 不同主题（default/cyber/elegant/ocean/sakura）在同一组件上的配色一致性提升。
  • 状态色（成功/警告/错误/信息）从分散 class 走向统一语义变量，后续维护成本降低。
  • 后续可逐步替换剩余硬编码而不动业务逻辑。

  • 进行中 继续收敛 BagPanel.vue 剩余少量视觉特效硬编码（阴影/渐变细节）。
  • 进行中 细化 FarmTools.vue iframe 注入样式中的个别绝对色（白字、toggle 默认色）。
  • 计划中 补充一份“主题验收清单”（浅色/深色/5主题）逐页面截图核对。
